## 시스템 개요

### 기자재 관리 시스템이란?

대학교 단과대학 및 학과별 기자재를 효율적으로 관리하는 웹 기반 시스템입니다.

**주요 기능**:
- ✅ 기자재 등록/조회/수정/삭제
- ✅ 단과대학/학과별 필터링
- ✅ 다중 선택 삭제
- ✅ 엑셀 다운로드
- ✅ 출력 기능 (목록/카드/상세)
- ✅ 사용자 권한 관리
- ✅ 동시 접속 지원 (최대 5명/계정)

### 시스템 요구사항

| 항목 | 요구사항 |
|------|----------|
| **운영체제** | Windows 10/11 (64bit) |
| **메모리** | 4GB 이상 |
| **저장공간** | 2GB 이상 |
| **네트워크** | 인터넷 연결 (최초 설치 시) |
| **권한** | 관리자 권한 (최초 설치 시) |
| **필수 소프트웨어** | WSL2 + Ubuntu 24.04 |

### 2단계: 배포 방법 선택

```
1. 공유 폴더 생성:
   \\공유서버\기자재관리\equipment_manager

2. equipment_manager 폴더 전체 복사

3. 팀원들에게 경로 안내:
   "\\공유서버\기자재관리\equipment_manager에서
    폴더를 다운받아 설치하세요"
```

#### 방법 B: USB 메모리

**장점**: 네트워크 불필요

```
1. USB에 equipment_manager 폴더 복사

2. 팀원들 PC를 방문하여 복사 지원

3. 설치 진행
```

#### 방법 C: 클라우드 공유 (OneDrive, Google Drive)

**장점**: 원격 배포 가능

```
1. OneDrive/Google Drive에 업로드

2. 공유 링크 생성

3. 팀원들에게 링크 전송
```

## 팀원 설치 방법

### 설치 전 준비사항

**체크리스트**:
- [ ] Windows 10/11 64bit 확인
- [ ] 관리자 권한 확보
- [ ] equipment_manager 폴더 다운로드
- [ ] 인터넷 연결 확인
- [ ] 30분 시간 확보

### 자동 설치 (추천) ⭐

#### 1단계: 폴더 복사

```
1. equipment_manager 폴더를 다음 위치에 복사:

   \\wsl.localhost\Ubuntu-24.04\home\[사용자이름]\

   예시:
   \\wsl.localhost\Ubuntu-24.04\home\kimcheolsu\equipment_manager
```

**참고**:
- `[사용자이름]`은 본인의 Windows 사용자 이름 또는 원하는 이름
- WSL이 설치되어 있지 않으면 다음 단계에서 자동 설치됨

#### 2단계: 자동 설치 스크립트 실행

```
1. equipment_manager 폴더 열기

2. "Setup-EquipmentManager.ps1" 파일 찾기

3. 우클릭 → "PowerShell에서 실행" 클릭

4. 안내 메시지에 따라 진행:
   - WSL 사용자명 입력 (기본값: Windows 사용자명)
   - WSL 미설치 시 자동 설치 (재부팅 필요)
   - Ubuntu 초기 설정 (사용자명/비밀번호 입력)

5. 설치 완료 메시지 확인

6. 바탕화면에 "기자재 관리" 아이콘 생성 확인
```

**예상 소요 시간**:
- WSL 이미 설치: 5분
- WSL 신규 설치: 20~30분 (재부팅 포함)

#### 3단계: 설치 확인

```
1. 바탕화면 "기자재 관리" 아이콘 더블클릭

2. 검은 창 열림 (서버 시작 중)

3. 3~5초 후 브라우저 자동 실행

4. http://localhost:8502 페이지 표시

5. 로그인 화면 확인
```

**성공 시**: 로그인 화면이 보입니다.
**실패 시**: [문제 해결](#문제-해결) 섹션 참조

### 수동 설치 (자동 설치 실패 시)

#### 1단계: WSL 설치

```
1. Windows + X → "Windows PowerShell (관리자)" 선택

2. 다음 명령어 입력:

   wsl --install -d Ubuntu-24.04

3. 재부팅

4. Ubuntu 초기 설정:
   - 사용자명 입력 (영문 소문자, 예: kimcheolsu)
   - 비밀번호 입력 (2회)
   - 비밀번호 기억할 것! (향후 사용)
```

#### 2단계: 폴더 복사

```
1. Windows 탐색기 열기 (Windows + E)

2. 주소창에 입력:

   \\wsl.localhost\Ubuntu-24.04\home\[사용자이름]\

   (Ubuntu 설정 시 입력한 사용자명 사용)

3. equipment_manager 폴더를 여기에 붙여넣기
```

#### 3단계: 바탕화면 아이콘 생성

```
1. equipment_manager 폴더 열기

2. "create-desktop-icon.vbs" 파일 더블클릭

3. WSL 사용자명 입력 (Ubuntu 설정 시 사용한 이름)

4. "바로가기 생성 완료" 메시지 확인

5. 바탕화면에 "기자재 관리" 아이콘 확인
```

#### 4단계: 설치 확인

자동 설치의 3단계와 동일

```
1. 기자재 조회 화면에서 목록 표시

4. 확인 팝업 → "✅ 삭제 확인" 클릭

5. "N개 품목이 삭제되었습니다" 메시지 확인
```

#### 4. 엑셀 저장

```
1. 기자재 조회 결과 표시

2. "📥 엑셀 저장" 버튼 클릭

3. "⬇️ 다운로드" 버튼 클릭
```

## 계정 관리

### 기본 관리자 계정

| 항목 | 값 |
|------|-----|
| **아이디** | admin |
| **비밀번호** | admin123 |
| **권한** | 전체 접근 |
| **용도** | 시스템 관리, 사용자 생성 |

**⚠️ 보안 권장사항**:
- 최초 로그인 후 비밀번호 변경
- 관리자 계정은 관리자만 사용
- 일반 업무는 부서 계정 사용

### 팀원 계정 생성 방법

```
1. 관리자 계정으로 로그인

2. "관리자 메뉴" 클릭

3. "사용자 관리" 클릭

4. "신규 사용자" 버튼 클릭

5. 정보 입력:
   - 사용자명: 부서코드 또는 부서명 (예: humanities)
   - 비밀번호: 부서별 설정 (8자 이상)
   - 역할: user (일반 사용자)
   - 단과대학: 해당 부서 선택
   - 이름: 담당자 이름 (선택)

6. "등록" 버튼 클릭

7. 팀원에게 계정 정보 전달:
   - 아이디: [입력한 사용자명]
   - 비밀번호: [설정한 비밀번호]
   - 접속 주소: http://localhost:8502
```

### 권한 관리

| 역할 | 권한 | 설명 |
|------|------|------|
| **admin** | 전체 | - 모든 단과대학 데이터 접근<br>- 사용자 관리<br>- 시스템 설정 |
| **user** | 제한적 | - 자신의 단과대학 데이터만 접근<br>- 기자재 등록/조회/수정<br>- 사용자 관리 불가 |

### 동시 접속 정책

```
한 계정으로 최대 5명까지 동시 접속 가능

예시:
- humanities 계정으로 인문대 담당자 3명이 동시 사용 가능
- 6번째 접속 시도 시 가장 오래된 세션 자동 종료

세션 유지 기간:
- 로그인 유지 체크: 3개월
- 로그인 유지 미체크: 브라우저 종료 시까지
- 30분 미사용 시 자동 로그아웃
```

## 문제 해결

### Q1. "이 페이지에 연결할 수 없습니다"

**원인**: 서버가 실행되지 않음

**해결**:
```
1. 바탕화면 "기자재 관리" 아이콘 더블클릭

2. 검은 창이 열리는지 확인

3. 3초 대기

4. 브라우저 새로고침 (F5)

5. 여전히 안 되면:
   - 검은 창 닫기
   - 아이콘 다시 실행
```

### Q2. "wsl 명령을 찾을 수 없습니다"

**원인**: WSL이 설치되지 않음

**해결**:
```
1. Windows PowerShell (관리자) 실행

2. 명령어 입력:
   wsl --install -d Ubuntu-24.04

3. 재부팅

4. Ubuntu 초기 설정 (사용자명/비밀번호)

5. 설치 스크립트 다시 실행
```

### Q3. "로그인할 수 없습니다"

**원인**: 계정 정보 오류 또는 미생성

**해결**:
```
1. 아이디/비밀번호 확인
   - 대소문자 구분
   - 공백 없는지 확인

2. 관리자에게 계정 생성 요청

3. 기본 관리자 계정으로 테스트:
   - ID: admin
   - PW: admin123
```

### Q4. "동시 접속 제한 초과"

**원인**: 같은 계정으로 5명 이상 접속 시도

**해결**:
```
1. 잠시 후 다시 시도 (30분 후 자동 정리)

2. 또는 관리자에게 세션 정리 요청

3. 근본 해결: 부서별 별도 계정 생성
```

### Q5. "검은 창이 바로 닫힘"

**원인**: 스크립트 오류 또는 경로 문제

**해결**:
```
1. equipment_manager 폴더 위치 확인:
   \\wsl.localhost\Ubuntu-24.04\home\[사용자명]\equipment_manager

2. start.sh 파일 존재 확인

3. WSL Ubuntu가 정상 작동하는지 확인:
   PowerShell에서 "wsl" 명령 실행
```

### A. 배포 시나리오별 가이드

#### 시나리오 1: 개인 PC 각각 실행

**적용 대상**: 소규모 팀 (3명 이하)

**특징**:
- ✅ 각자 독립적 사용
- ✅ 서로 영향 없음
- ❌ 데이터 분산 (통합 관리 어려움)
- ❌ 백업 각자 관리 필요

**설치 방법**:
```
1. 각 팀원 PC에 equipment_manager 설치

2. 각자 바탕화면 아이콘으로 실행

3. 각자 데이터 관리
```

**데이터 백업**:
```
정기적으로 백업 필요:
equipment_manager/data/equipment.db → USB 또는 네트워크 저장
```

#### 시나리오 2: 공용 서버 1대 실행 (추천)

**적용 대상**: 중대규모 팀 (4명 이상)

**특징**:
- ✅ 데이터 통합 관리
- ✅ 백업 용이
- ✅ 동시 접속 가능
- ❌ 서버 PC 꺼지면 전체 중단

**설정 방법**:

1. **서버 PC 지정**:
```
- 항상 켜져있는 PC 선택
- 또는 전용 서버 PC 준비
```

2. **서버 PC에만 설치**:
```
equipment_manager 폴더 설치
바탕화면 아이콘으로 서버 시작
```

3. **서버 PC IP 확인**:
```
PowerShell에서:
ipconfig

IPv4 주소 확인 (예: 192.168.1.100)
```

4. **팀원들 설정**:
```
각자 PC에서 브라우저 북마크:
http://192.168.1.100:8502

(서버 PC IP 주소 사용)
```

5. **서버 PC 자동 시작** (선택):
```
Windows 작업 스케줄러:
- 부팅 시 EquipmentManager.bat 실행
```

**장점**:
- 데이터 중앙 관리
- 한 곳에서 백업
- 업데이트 용이

**주의사항**:
- 서버 PC는 항상 켜두기
- 정기적 백업 필수
- 서버 PC 성능 충분해야 함

### C. 업데이트 배포 절차

**관리자 작업**:
```
1. 새 버전 equipment_manager 폴더 준비

2. 공유 폴더 또는 USB에 복사

3. 팀원들에게 업데이트 안내:
   - 업데이트 내용
   - 다운로드 경로
   - 업데이트 방법
```

**팀원 작업**:
```
1. 서버 종료 (검은 창 닫기)

2. 기존 equipment_manager 폴더 백업:
   equipment_manager → equipment_manager_backup

3. data 폴더만 따로 보관:
   equipment_manager_backup/data 복사

4. 새 equipment_manager 폴더 다운로드

5. 기존 data 폴더를 새 폴더에 복사:
   equipment_manager/data ← (기존 data)

6. 바탕화면 아이콘 다시 실행

7. 정상 작동 확인
```

### D. 데이터 백업 및 복구

```
1. 서버 종료

2. 다음 폴더 복사:
   equipment_manager/data/

3. 백업 저장 위치:
   - 외장 하드: D:\백업\기자재관리\[날짜]
   - 네트워크: \\서버\백업\기자재관리\[날짜]
   - 클라우드: OneDrive, Google Drive 등

4. 압축 파일로 저장 (선택):
   data_20260327.zip
```

```

2. 현재 data 폴더 삭제 또는 이름 변경

   백업\data → equipment_manager\data

5. 데이터 확인
```
