## 개요

### 서버 PC 방식의 장점

✅ **데이터 중앙 집중화**
- 모든 데이터가 한 곳에 저장
- 통합 관리 및 백업 용이

✅ **동시 접속 지원**
- 여러 팀원이 동시에 사용 가능
- 데이터 실시간 공유

✅ **효율적 리소스 활용**
- 안 쓰는 PC를 서버로 재활용
- 별도 서버 구매 불필요

✅ **업데이트 관리 간편**
- 서버 PC 한 곳만 업데이트
- 클라이언트 PC는 업데이트 불필요

## 1단계: 서버 PC 기본 설정

### 1-1. Windows 업데이트

```

2. "업데이트 확인" 클릭

3. 모든 업데이트 설치

```


### 1-3. 전원 옵션 설정

**중요**: 서버가 꺼지지 않도록 설정

```

2. "화면 끄기": 30분

3. "절전 모드": 안 함

   - 절전 모드: 안 함

```

## 2단계: 네트워크 설정

### 2-1. 고정 IP 설정 (매우 중요!)

**왜 필요한가?**
- IP가 바뀌면 팀원들이 접속 불가
- 고정 IP로 설정하여 항상 같은 주소 유지

**설정 방법**:

```
1. 설정 → 네트워크 및 인터넷 → 이더넷

2. 현재 연결된 네트워크 클릭

3. "IP 설정": 편집 클릭

4. "자동(DHCP)" → "수동" 변경

5. IPv4 켜기

6. 정보 입력:
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   IP 주소:    192.168.1.100
   서브넷 마스크: 255.255.255.0
   게이트웨이:   192.168.1.1

   기본 DNS:    192.168.1.1
   보조 DNS:    8.8.8.8
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   ⚠️ 주의: 위 값은 예시입니다!
   실제 네트워크 환경에 맞게 수정하세요.

7. "저장" 클릭
```

**실제 값 확인 방법**:

```
1. Windows + R → cmd

2. ipconfig 입력

3. 현재 값 확인:
   - IPv4 주소: 192.168.X.X
   - 서브넷 마스크: 255.255.255.0
   - 기본 게이트웨이: 192.168.X.1

4. 위 값을 참고하여 고정 IP 설정
   (끝자리만 100으로 변경)
```

## 3단계: 소프트웨어 설치

### 3-1. WSL 설치

```
1. Windows PowerShell (관리자) 실행

2. 명령어 입력:

   wsl --install -d Ubuntu-24.04

3. 재부팅

4. Ubuntu 초기 설정:
   - 사용자명: server (권장)
   - 비밀번호: [안전한 비밀번호]

   ⚠️ 비밀번호 반드시 기록!
```

### 3-2. Equipment Manager 설치

**방법 A: 공유 폴더에서 복사**

```
1. 개발 PC에서 equipment_manager 폴더 공유

2. 서버 PC에서 공유 폴더 접속:
   \\개발PC\공유폴더\equipment_manager

3. 폴더 전체 복사

4. 붙여넣기 위치:
   \\wsl.localhost\Ubuntu-24.04\home\server\

5. 최종 경로:
   \\wsl.localhost\Ubuntu-24.04\home\server\equipment_manager
```

**방법 B: USB로 복사**

```
1. USB에 equipment_manager 폴더 복사

2. 서버 PC에 USB 연결

3. USB의 equipment_manager 폴더 복사

4. 붙여넣기:
   \\wsl.localhost\Ubuntu-24.04\home\server\
```

### 3-3. 설치 확인

```
1. Windows PowerShell 실행

2. WSL 진입:
   wsl -d Ubuntu-24.04

3. 폴더 확인:
   cd /home/server/equipment_manager
   ls -la

4. 필수 파일 확인:
   - start.sh ✓
   - stop.sh ✓
   - main.py ✓
   - data/ 폴더 ✓

5. 권한 설정:
   chmod +x start.sh
   chmod +x stop.sh

6. WSL 종료:
   exit
```

## 4단계: 부팅 시 자동 시작 설정

### 4-1. 자동 시작 스크립트 생성

이미 생성된 스크립트를 사용하거나, 새로 만듭니다:

**파일 위치**:
```
\\wsl.localhost\Ubuntu-24.04\home\server\equipment_manager\ServerAutoStart.bat
```

**내용**:
```batch
@echo off
REM Equipment Manager - Server Auto Start

title Equipment Manager Server

REM Wait for network
timeout /t 30 /nobreak > nul

REM Start server
cd /d \\wsl.localhost\Ubuntu-24.04\home\server\equipment_manager
start /B wsl -d Ubuntu-24.04 -e bash -c "cd /home/server/equipment_manager && nohup ./start.sh > /dev/null 2>&1 &"

REM Wait for startup
timeout /t 10 /nobreak > nul

echo Server started successfully!
echo Access: http://localhost:8502
echo.

REM Minimize window
powershell -command "$wshell = New-Object -ComObject WScript.Shell; $wshell.SendKeys('% n')"
```

## 7단계: 백업 자동화

### 7-1. 백업 스크립트 생성

**파일 위치**:
```
\\wsl.localhost\Ubuntu-24.04\home\server\equipment_manager\BackupData.bat
```

**내용**:
```batch
@echo off
REM Equipment Manager - Daily Backup Script

set BACKUP_DIR=D:\Backups\EquipmentManager
set DATE=%date:~0,4%%date:~5,2%%date:~8,2%

REM Create backup directory
if not exist "%BACKUP_DIR%" mkdir "%BACKUP_DIR%"
if not exist "%BACKUP_DIR%\%DATE%" mkdir "%BACKUP_DIR%\%DATE%"

REM Copy data folder
xcopy /E /I /Y "\\wsl.localhost\Ubuntu-24.04\home\server\equipment_manager\data" "%BACKUP_DIR%\%DATE%\data"

REM Create ZIP archive
powershell Compress-Archive -Path "%BACKUP_DIR%\%DATE%\*" -DestinationPath "%BACKUP_DIR%\backup_%DATE%.zip" -Force

REM Delete old backups (keep last 30 days)
forfiles /P "%BACKUP_DIR%" /M *.zip /D -30 /C "cmd /c del @path" 2>nul

echo Backup completed: %BACKUP_DIR%\backup_%DATE%.zip
```

### 7-2. 작업 스케줄러로 자동 백업 설정


## 문제 해결

### Q1. 서버가 부팅 후 자동 시작되지 않음

**확인 사항**:
```
1. 작업 스케줄러에서 작업 확인:
   - 상태: "실행 중" 또는 "준비"
   - 마지막 실행 결과: 0x0 (성공)

2. 작업 우클릭 → "실행" 클릭하여 수동 실행 테스트

3. 작업 속성 → 기록 탭에서 오류 확인
```

### Q2. 팀원 PC에서 접속 불가

**체크리스트**:
```
1. 서버 PC에서 접속 테스트:
   http://localhost:8502 → 성공하면 서버는 정상

2. 같은 네트워크 확인:
   서버 PC와 클라이언트 PC가 같은 네트워크에 있는지

3. IP 주소 확인:
   서버 PC에서 ipconfig → IP 변경되었는지

4. 방화벽 확인:
   포트 8502 개방되었는지

5. Ping 테스트:
   cmd에서 ping 192.168.1.100 → 응답 확인
```

### Q3. 서버가 느려짐

**해결 방법**:
```
1. 작업 관리자 → 성능 탭:
   - CPU 사용률 확인
   - 메모리 사용률 확인
   - 디스크 사용률 확인

2. 데이터베이스 최적화:
   - 불필요한 데이터 삭제
   - 인덱스 재구성

3. 로그 파일 정리:
   - 오래된 로그 삭제

4. 서버 재시작:
   - 정기적 재시작 (주 1회 권장)
```

### Q4. 백업이 실행되지 않음

**확인**:
```
1. 백업 폴더 경로 확인:
   D:\Backups\EquipmentManager 존재하는지

2. 권한 확인:
   작업 스케줄러 작업이 충분한 권한으로 실행되는지

3. 수동 실행 테스트:
   BackupData.bat 더블클릭하여 수동 실행
```
