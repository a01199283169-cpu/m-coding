## 6. Architecture Considerations

### 6.1 Project Level Selection

| Level | Characteristics | Recommended For | Selected |
|-------|-----------------|-----------------|:--------:|
| **Starter** | Simple structure (`components/`, `lib/`, `types/`) | Static sites, portfolios, landing pages | ☑ |
| **Dynamic** | Feature-based modules, BaaS integration (bkend.ai) | Web apps with backend, SaaS MVPs, fullstack apps | ☐ |
| **Enterprise** | Strict layer separation, DI, microservices | High-traffic systems, complex architectures | ☐ |

> **Rationale**: Marp 슬라이드는 정적 문서이므로 Starter 레벨 적용

### 6.2 Key Architectural Decisions

| Decision | Options | Selected | Rationale |
|----------|---------|----------|-----------|
| 문서 형식 | Marp / PowerPoint / PDF | Marp | 기존 형식 유지, 버전 관리 용이 |
| 테마 | default / gaia / uncover | default | 원본과 동일 |
| 출력 방식 | HTML / PDF / PPTX | HTML | 기존 방식 유지 |

### 6.3 Clean Architecture Approach

```
Selected Level: Starter

Folder Structure:
ummu/

  ├── 업무중심체제_전환계획안_marp.md (원본 슬라이드)
  ├── 업무중심체제_전환계획안_marp.html (빌드 결과)
```
