## 1. Overview

### 1.1 Purpose

직원들이 휴가(년가, 지참, 조퇴, 외출, 병가, 공가)를 신청하고, 신청 내역을 조회할 수 있는 웹 애플리케이션을 개발합니다. 캘린더 형태로 휴가 일정을 시각화하여 사용자가 직관적으로 휴가 현황을 파악할 수 있도록 합니다.

### 1.2 Background

- 수기 또는 스프레드시트로 관리하던 휴가 신청 프로세스를 디지털화
- 개인 학습 및 포트폴리오 목적의 Starter 레벨 프로젝트
- Python 백엔드와 bkend.ai BaaS를 활용한 간단한 풀스택 구조

### 1.3 Related Documents

- Requirements: 사용자 요구사항 (본 문서 내 정의)
- References: bkend.ai 문서, Python Flask/FastAPI 문서

## 2. Scope

### 2.1 In Scope

- [x] **휴가 종류**: 년가, 지참, 조퇴, 외출, 병가, 공가
- [x] **CRUD 기능**: 휴가 신청 생성, 조회, 수정, 삭제
- [x] **캘린더 뷰**: 월별/주별 캘린더에 휴가 일정 표시
- [x] **사용자 인증**: bkend.ai BaaS 기반 간단한 로그인/회원가입
- [x] **휴가 목록**: 본인이 신청한 휴가 리스트 조회

### 2.2 Out of Scope

- 승인 워크플로우 (관리자 승인 기능)
- 이메일/푸시 알림
- 잔여 휴가 일수 계산 (단순 신청만)
- 팀/조직 관리 기능
- 모바일 앱 (웹 브라우저 접근만)

## 6. Architecture Considerations

### 6.1 Project Level Selection

| Level | Characteristics | Recommended For | Selected |
|-------|-----------------|-----------------|:--------:|
| **Starter** | Simple structure (`components/`, `lib/`, `types/`) | Static sites, portfolios, landing pages | ☑ |
| **Dynamic** | Feature-based modules, BaaS integration (bkend.ai) | Web apps with backend, SaaS MVPs, fullstack apps | ☐ |
| **Enterprise** | Strict layer separation, DI, microservices | High-traffic systems, complex architectures | ☐ |

**선택 이유**: 개인 학습용 프로젝트이며, 단순한 CRUD와 캘린더 뷰만 제공하므로 Starter 레벨이 적합합니다.

### 6.2 Key Architectural Decisions

| Decision | Options | Selected | Rationale |
|----------|---------|----------|-----------|
| Backend Framework | Flask / FastAPI / Django | **FastAPI** | 빠르고 간결한 API 개발, 자동 문서화 지원 |
| Frontend | HTML/Jinja2 / React / Vue | **HTML + Jinja2** | Starter 레벨에 적합한 간단한 템플릿 방식 |
| Database | bkend.ai BaaS / SQLite / PostgreSQL | **bkend.ai BaaS** | 백엔드 인프라 관리 불필요, 인증 기능 포함 |
| Styling | Tailwind / Bootstrap / Plain CSS | **Tailwind CSS** | 빠른 스타일링, 반응형 지원 |
| Calendar Library | FullCalendar / Custom HTML | **Custom HTML/CSS** | 외부 의존성 최소화, 학습 목적 |
| Deployment | Vercel / Heroku / Local | **Local** | Starter 레벨, 프로덕션 배포 아님 |
