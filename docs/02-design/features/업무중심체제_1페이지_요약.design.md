## 1. Overview

### 1.1 Design Goals

- **간결성**: A4 1장 출력 시 모든 내용이 한눈에 보이도록 설계
- **명확성**: 핵심 메시지 6가지가 명확하게 구분되도록 배치
- **일관성**: 원본 슬라이드와 동일한 스타일 및 브랜딩 유지
- **가독성**: 글꼴 크기와 여백을 조정하여 출력물 가독성 확보

### 1.2 Design Principles

- **정보 우선순위**: 가장 중요한 정보(배경, 문제점, 방안)를 상단에 배치
- **시각적 단순성**: 불필요한 장식 제거, 핵심 내용에 집중
- **Marp 표준 준수**: Marp 문법을 준수하여 HTML 빌드 보장

## 3. 내용 배치 계획

### 3.1 핵심 메시지 6가지 요약

| 순번 | 원본 섹션 | 요약 내용 | 글자 수 제한 |
|------|----------|----------|------------|
| 1 | 현행 체계 | 학과담당제 운영 (22명, 3그룹, 6개 대학) | 30자 |
| 2 | 문제점 | 전문성 분산, 업무 연속성 취약, 효율성 저하 | 40자 |
| 3 | 전환 방안 | 6개 업무 분야별 전담 체제 (학사/교무/학생/교직/현장실습/기타) | 50자 |
| 4 | 기대 효과 | 전문성 강화, 업무 연속성 확보, 민원 품질 향상 | 40자 |
| 5 | 추진 일정 | 2026년 3-4월 협의, 상반기 매뉴얼 구축 | 30자 |
| 6 | 협조 사항 | 공석 3명 조기 충원, 외부 연수 지원 | 30자 |

**총 글자 수**: 약 220자 (원본 대비 90% 축소)

### 3.2 표 최소화

원본에 있는 표 중 필수만 포함:

| 원본 표 | 포함 여부 | 간소화 방법 |
|---------|----------|-----------|
| 그룹별 인원 현황 | ❌ 제외 | 텍스트로 간단히 설명 (22명, 3그룹) |
| 업무 분야 6개 | ✅ 포함 | 나열 형식 (학사/교무/학생/교직/현장실습/기타) |
| 그룹별 업무 분장 | ❌ 제외 | 불필요 (상세본 참조) |
| 추진 일정 | ✅ 포함 | 핵심 마일스톤만 (3-4월 협의, 상반기 완료) |

### 3.3 이미지 제거

- 원본의 `./aaa.png` (업무분장표) 이미지 제외
- 필요시 상세본 참조 유도

## 4. 스타일 정의

### 4.1 Marp 프론트매터

원본 스타일을 그대로 사용하되, 글꼴 크기만 조정:

```yaml
---
marp: true
theme: default
paginate: false  # 1페이지이므로 페이지 번호 불필요
header: '업무중심 체제 전환 계획안 (요약)'
footer: '신한대학교 교무처 행정1팀 | 2026. 3. 19.'
style: |
  section {
    font-family: 'Noto Sans KR', 'Malgun Gothic', 'Apple SD Gothic Neo', sans-serif;
    font-size: 20px;  # 원본 26px → 20px (압축)
  }
  h1 {
    color: #003d82;
    border-bottom: 3px solid #003d82;
    padding-bottom: 10px;
    font-size: 32px;  # 제목 크기 유지
  }
  h2 {
    color: #0066cc;
    font-size: 24px;  # 소제목 크기 축소
  }
  ul, ol {
    font-size: 18px;  # 목록 글꼴 축소
  }
---
```

### 4.2 글꼴 크기 조정 전략

| 요소 | 원본 크기 | 요약본 크기 | 비고 |
|------|----------|-----------|------|
| 본문 (section) | 26px | 20px | -23% 축소 |
| 제목 (h1) | auto | 32px | 유지 |
| 소제목 (h2) | auto | 24px | 축소 |
| 목록 (ul, ol) | 22px | 18px | -18% 축소 |

## 5. Marp 문법 가이드

### 5.1 슬라이드 구분

```markdown
---
marp: true
...
---

<!-- 슬라이드 1 -->
# 제목

내용

---

<!-- 슬라이드 2 (필요 시) -->
# 제목

내용
```

### 5.2 주석 및 클래스

- `<!-- _class: lead -->`: 표지 슬라이드 (중앙 정렬)
- `<!-- _paginate: false -->`: 페이지 번호 숨김
- `<!-- _header: "" -->`: 특정 슬라이드에서 헤더 제거

## 8. 에러 처리

### 8.1 예상 문제 및 해결책

| 문제 | 원인 | 해결책 |
|------|------|--------|
| 내용이 1장을 초과 | 텍스트 과다 | 글꼴 크기 추가 축소 (18px) 또는 내용 간소화 |
| Marp 빌드 실패 | 문법 오류 | YAML 프론트매터 검증, 마크다운 문법 확인 |
| 스타일 미적용 | CSS 오류 | style 블록 내 세미콜론(;) 확인 |
| 한글 깨짐 | 폰트 미지원 | 시스템 폰트 fallback 확인 |
