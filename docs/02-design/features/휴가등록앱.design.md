## 1. Overview

### 1.1 Design Goals

1. **간단하고 직관적인 휴가 신청 프로세스**: 최소한의 입력으로 휴가 등록 가능
2. **캘린더 기반 시각화**: 사용자가 한눈에 휴가 일정을 파악할 수 있도록 함
3. **bkend.ai BaaS 활용**: 백엔드 인프라 관리 없이 빠른 개발 및 배포
4. **반응형 웹 디자인**: 모바일/데스크톱 모두에서 사용 가능

### 1.2 Design Principles

- **단순성 (Simplicity)**: 불필요한 기능 제거, 핵심 CRUD만 구현
- **보안성 (Security)**: bkend.ai 인증 기반 접근 제어, 본인 데이터만 조회 가능
- **유지보수성 (Maintainability)**: PEP8 준수, 명확한 함수/변수 네이밍
- **확장 가능성 (Extensibility)**: 향후 승인 워크플로우 추가 가능한 구조

## 3. Data Model

### 3.1 Entity Definition (Python)

```python
# models.py
from pydantic import BaseModel, Field
from datetime import date
from enum import Enum

class LeaveType(str, Enum):
    """휴가 종류"""
    ANNUAL = "년가"       # Annual Leave
    HALF_DAY = "지참"     # Half-day Leave
    EARLY_LEAVE = "조퇴"  # Early Leave
    OUTING = "외출"       # Outing
    SICK = "병가"         # Sick Leave
    PUBLIC = "공가"       # Public/Official Leave

class Leave(BaseModel):
    """휴가 엔티티"""
    id: str | None = None                    # bkend.ai ObjectId
    user_id: str                             # 신청자 ID (bkend.ai user)
    leave_type: LeaveType                    # 휴가 종류
    start_date: date                         # 시작일
    end_date: date                           # 종료일
    reason: str | None = Field(max_length=500, default=None)  # 사유 (선택)
    created_at: str | None = None           # 생성 시간 (bkend.ai auto)
    updated_at: str | None = None           # 수정 시간 (bkend.ai auto)

    class Config:
        json_schema_extra = {
            "example": {
                "user_id": "user123",
                "leave_type": "년가",
                "start_date": "2026-02-15",
                "end_date": "2026-02-17",
                "reason": "가족 여행"
            }
        }

class LeaveCreateRequest(BaseModel):
    """휴가 신청 요청"""
    leave_type: LeaveType
    start_date: date
    end_date: date
    reason: str | None = Field(max_length=500, default=None)

class LeaveUpdateRequest(BaseModel):
    """휴가 수정 요청"""
    leave_type: LeaveType | None = None
    start_date: date | None = None
    end_date: date | None = None
    reason: str | None = Field(max_length=500, default=None)
```

### 3.2 Entity Relationships

```
[User (bkend.ai)] 1 ──── N [Leave]
       │
       └── Authentication/Authorization (bkend.ai 제공)
```

- **User**: bkend.ai 인증 시스템에서 관리 (별도 테이블 불필요)
- **Leave**: 사용자별 휴가 신청 데이터 (본 프로젝트에서 생성)

### 3.3 MongoDB Collection Schema (bkend.ai)

**Collection Name**: `leaves`

| Field | Type | Required | Unique | Description |
|-------|------|----------|--------|-------------|
| _id | ObjectId | auto | auto | bkend.ai 자동 생성 |
| user_id | String | Yes | No | 신청자 ID (bkend.ai user) |
| leave_type | String (Enum) | Yes | No | 년가/지참/조퇴/외출/병가/공가 |
| start_date | Date | Yes | No | 휴가 시작일 (YYYY-MM-DD) |
| end_date | Date | Yes | No | 휴가 종료일 (YYYY-MM-DD) |
| reason | String | No | No | 휴가 사유 (최대 500자) |
| createdBy | String | auto | - | bkend.ai 자동 기록 |
| createdAt | Date | auto | - | bkend.ai 자동 기록 |
| updatedAt | Date | auto | - | bkend.ai 자동 기록 |

**Validation Rules**:
- `start_date` ≤ `end_date`
- `leave_type` must be one of 6 predefined values
- `user_id` must exist in bkend.ai user table

**Indexes**:
- `user_id` (non-unique) - 빠른 사용자별 조회
- `start_date`, `end_date` (compound) - 날짜 범위 검색 최적화

### 4.3 Detailed Specification

#### `POST /leaves` (휴가 신청 생성)

**Request (Form Data or JSON):**
```json
{
  "leave_type": "년가",
  "start_date": "2026-02-15",
  "end_date": "2026-02-17",
  "reason": "가족 여행"
}
```

**Response (201 Created):**
```json
{
  "id": "65f1a2b3c4d5e6f7g8h9i0j1",
  "user_id": "user123",
  "leave_type": "년가",
  "start_date": "2026-02-15",
  "end_date": "2026-02-17",
  "reason": "가족 여행",
  "createdAt": "2026-02-12T10:30:00Z",
  "updatedAt": "2026-02-12T10:30:00Z"
}
```

**Error Responses:**
- `400 Bad Request`: 입력 검증 실패 (날짜 형식 오류, start_date > end_date)
- `401 Unauthorized`: 인증 필요
- `422 Unprocessable Entity`: Pydantic 검증 실패

#### `GET /leaves` (본인 휴가 목록 조회)

**Query Parameters:**
- `month`: 조회 월 (YYYY-MM, 선택)
- `limit`: 결과 개수 제한 (기본값: 50)
- `offset`: 페이지네이션 오프셋 (기본값: 0)

**Response (200 OK):**
```json
{
  "items": [
    {
      "id": "...",
      "leave_type": "년가",
      "start_date": "2026-02-15",
      "end_date": "2026-02-17",
      "reason": "가족 여행",
      "createdAt": "..."
    }
  ],
  "total": 10,
  "limit": 50,
  "offset": 0
}
```

#### `POST /leaves/{id}/delete` (휴가 삭제)

**Response (204 No Content):**
- 성공 시 빈 응답

**Error Responses:**
- `403 Forbidden`: 본인 휴가가 아님
- `404 Not Found`: 해당 휴가 없음

## 6. Error Handling

### 6.1 Error Code Definition

| Code | Message | Cause | Handling |
|------|---------|-------|----------|
| 400 | 입력 데이터가 올바르지 않습니다 | 날짜 형식 오류, start_date > end_date | 폼에 에러 메시지 표시 |
| 401 | 로그인이 필요합니다 | 인증 토큰 없음/만료 | 로그인 페이지로 리다이렉트 |
| 403 | 권한이 없습니다 | 타인의 휴가 수정/삭제 시도 | 에러 페이지 표시 |
| 404 | 휴가를 찾을 수 없습니다 | 삭제된 휴가 또는 잘못된 ID | 목록 페이지로 리다이렉트 |
| 422 | 입력 검증 실패 | Pydantic 모델 검증 실패 | 상세 에러 메시지 표시 |
| 500 | 서버 오류가 발생했습니다 | bkend.ai API 오류, 서버 예외 | 로그 기록, 일반 에러 페이지 |

### 6.2 Error Response Format (JSON API)

```json
{
  "error": {
    "code": "INVALID_INPUT",
    "message": "시작일은 종료일보다 이전이어야 합니다",
    "details": {
      "field": "start_date",
      "value": "2026-02-20"
    }
  }
}
```

### 6.3 Error Handling Pattern (FastAPI)

```python
# routes/leaves.py
from fastapi import HTTPException

@app.post("/leaves")
async def create_leave(leave: LeaveCreateRequest):
    try:
        # 날짜 검증
        if leave.start_date > leave.end_date:
            raise HTTPException(
                status_code=400,
                detail="시작일은 종료일보다 이전이어야 합니다"
            )

        # bkend.ai API 호출
        result = await bkend_service.create_leave(leave)
        return result

    except HTTPException:
        raise
    except Exception as e:
        logger.error(f"휴가 생성 오류: {e}")
        raise HTTPException(
            status_code=500,
            detail="서버 오류가 발생했습니다"
        )
```

## 7. Security Considerations

- [x] **Input validation**: Pydantic 모델로 입력 검증, XSS 방지를 위한 HTML 이스케이프
- [x] **Authentication**: bkend.ai JWT 토큰 기반 인증
- [x] **Authorization**: RLS (Row Level Security) - 본인 데이터만 조회/수정/삭제 가능
- [x] **HTTPS enforcement**: 프로덕션 환경에서 HTTPS 강제 (개발 환경 제외)
- [x] **CSRF protection**: FastAPI CSRF 미들웨어 적용 (POST 요청)
- [ ] Rate Limiting: 향후 추가 (FastAPI Limiter)
- [x] **Sensitive data encryption**: bkend.ai에서 암호화 처리, 비밀번호 해싱

### 7.1 RLS (Row Level Security) Policy

bkend.ai에서 제공하는 RLS를 통해 사용자별 데이터 격리:

```python
# bkend.ai RLS 설정 (MCP 또는 대시보드에서)
# Policy: user_id = current_user_id()
#
# 효과:
# - 사용자는 자신의 user_id와 일치하는 휴가만 조회 가능
# - 타인의 휴가 수정/삭제 시도 시 403 Forbidden 반환
```

## 10. Coding Convention

> Reference: PEP8 + FastAPI Best Practices

### 10.1 Naming Conventions

| Target | Rule | Example |
|--------|------|---------|
| Functions | snake_case | `get_user_leaves()`, `create_leave()` |
| Classes | PascalCase | `Leave`, `LeaveType` |
| Constants | UPPER_SNAKE_CASE | `MAX_REASON_LENGTH`, `API_BASE_URL` |
| Variables | snake_case | `user_id`, `start_date` |
| Files | snake_case.py | `leaves.py`, `bkend.py` |
| Templates | snake_case.html | `new_leave.html`, `calendar.html` |
| Folders | snake_case | `routes/`, `services/` |

### 10.2 Import Order

```python
# 1. 표준 라이브러리
from datetime import date, datetime
from enum import Enum

# 2. 서드파티 라이브러리
from fastapi import FastAPI, HTTPException, Request
from pydantic import BaseModel, Field

# 3. 로컬 모듈
from app.services.bkend import BkendService
from app.models import Leave, LeaveType
```

### 10.3 Environment Variables

| Variable | Purpose | Scope | Example |
|----------|---------|-------|---------|
| `BKEND_API_KEY` | bkend.ai 인증 키 | Server | `bkapi_xxxxx` |
| `BKEND_PROJECT_ID` | bkend.ai 프로젝트 ID | Server | `proj_xxxxx` |
| `BKEND_ENV_ID` | bkend.ai 환경 ID | Server | `env_production` |
| `SECRET_KEY` | FastAPI 세션 암호화 | Server | `random-secret-key` |
| `DEBUG` | 디버그 모드 | Server | `True` / `False` |

### 10.4 Code Style Rules

```python
# ✅ Good
def get_user_leaves(user_id: str, month: str | None = None) -> list[Leave]:
    """사용자의 휴가 목록을 조회합니다.

    Args:
        user_id: 사용자 ID
        month: 조회할 월 (YYYY-MM 형식, 선택)

    Returns:
        휴가 객체 리스트
    """
    # 구현...

# ❌ Bad
def getUserLeaves(userId,month=None):  # camelCase 사용, 타입 힌트 없음
    # 구현...
```

### 10.5 Error Handling Convention

```python
# ✅ Good: 명확한 에러 메시지, 로그 기록
try:
    leave = await bkend_service.get_leave(leave_id)
except LeaveNotFoundError:
    logger.warning(f"Leave not found: {leave_id}")
    raise HTTPException(status_code=404, detail="휴가를 찾을 수 없습니다")
except Exception as e:
    logger.error(f"Unexpected error: {e}")
    raise HTTPException(status_code=500, detail="서버 오류가 발생했습니다")

# ❌ Bad: 일반적인 Exception만 캐치, 로그 없음
try:
    leave = await bkend_service.get_leave(leave_id)
except:
    raise HTTPException(status_code=500, detail="Error")
```

## 11. Implementation Guide

### 11.1 Implementation Order

1. [ ] **환경 설정**
   - Python 가상환경 생성
   - `requirements.txt` 작성 및 설치 (fastapi, uvicorn, jinja2, python-dotenv, httpx)
   - `.env` 파일 생성 (BKEND_API_KEY, BKEND_PROJECT_ID 등)

2. [ ] **bkend.ai 프로젝트 설정**
   - bkend.ai 대시보드에서 프로젝트 생성
   - MCP 연결 (Claude Code에서 bkend.ai MCP 설정)
   - `leaves` 테이블 생성 (6가지 휴가 종류 지원)

3. [ ] **데이터 모델 정의**
   - `app/models.py` 작성 (Leave, LeaveType, Pydantic 모델)
   - 날짜 검증 로직 추가 (start_date ≤ end_date)

4. [ ] **bkend.ai 서비스 레이어**
   - `app/services/bkend.py` 작성
   - bkend.ai REST API 클라이언트 구현 (CRUD 메서드)

5. [ ] **API 라우트 구현**
   - `app/routes/auth.py` (로그인/회원가입)
   - `app/routes/leaves.py` (휴가 CRUD)

6. [ ] **HTML 템플릿 작성**
   - `templates/base.html` (Tailwind CSS CDN 포함)
   - `templates/login.html`, `templates/signup.html`
   - `templates/calendar.html`, `templates/list.html`
   - `templates/new_leave.html`, `templates/edit_leave.html`

7. [ ] **캘린더 UI 구현**
   - `static/js/calendar.js` (월별 캘린더 렌더링)
   - CSS로 휴가 종류별 색상 구분

8. [ ] **통합 테스트**
   - 수동 테스트: 로그인 → 휴가 신청 → 캘린더 확인
   - 에러 케이스 테스트 (401, 403, 404)

9. [ ] **코드 품질 검증**
   - Flake8 린트 검사
   - Black 포맷팅 적용
